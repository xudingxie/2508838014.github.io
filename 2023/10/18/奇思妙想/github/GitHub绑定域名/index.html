GitHub绑定域名

今天突发奇想把github的博客绑定好上自己的域名。下面是一些步骤和遇到的问题

步骤：

  1. 记录ipv4的信息。
  2. 将域名绑定到原本的github的page的地址
  3. 在github的pages中绑定域名。

问题

不满足启用https的条件

enforce https为灰色

提示：There isn’t a GitHub Pages site here. If you’re trying to publish one, read the full documentation to learn how to set up GitHub Pages for your repository, organization, or user account.

解决方案：

  1. cloundflare中开启了tls。将tls关闭。删除github中绑定的域名。重新添加。
  2. 该方案来自网络。此处仅仅是收集做参考。可以删除域名之后，重新添加并且绑定。即可。

绑定成功但是页面没有显示。直接为空。

我在操作的时候开始是没有按照步骤完成的。首先进行了操作2.并没有进行操作1就会导致这个问题。

绑定成功访问并没有访问到我的页面。但是界面中已经出现github的404.

这个我没有遇到，但是此处给一个排查的思路。

我是在GitHub的绑定域名已经成功，但是此时还没有启用https。此时我如果在github的界面中进行删除绑定的域名就会出现这个问题。
