C#Sharp-SQL-Server的连接池
发表于2023-05-212024-05-09 sql server
连接池实际上都是由sqlServer进行管理的。并且现在都是默认启动了。最小0，最大100.可以修改这个默认配置。刚才其实是在搞如何实现内容。刚刚在项如果我在同一个程序中有实例化了多个数据库套接字，创建了多个连接，而不在相同的连接池。那样岂不是蠢，白白占用了更多的资源（连接池的最小数目*创建的套接字）。所以目前的想法,是公用同一个套接字。那怎么实现，静态变量可以。然后分散在多个程序中。其中还有可以注意的。这里其实dispose与close是差不多的。释放只是多了把中间的连接字符串清空，它也是返回了连接池，close，是直接返回连接池。如果真的是不返回连接池的话，那岂不是很多就是没有用的了。所以还是一致的。这个是从别人文章中看到的。不知道现在还是不是。毕竟现在这些还是不好说的内容。那个文档还是2013年的了。wen
