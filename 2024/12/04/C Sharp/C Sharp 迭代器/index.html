迭代器

平时使用中经常看到IEnumerable与IEnumerator这两个在遍历时发挥作用。

但是对于它们却一无所知。仅仅知道IEnumerable时List，Array这类的接口。可以通过IEnumerable来实现彼此转换或者说更加自由的输入与输出。

它们一起提供了一种在集合上进行循环迭代的通用方式。

IEnumerable 接口

IEnumerable 接口定义了一种方法，用于返回一个实现了 IEnumerator 接口的迭代器。它有一个方法：IEnumerable 接口定义了一种方法，用于返回一个实现了 IEnumerator 接口的迭代器。它有一个方法：

GetEnumerator() 方法返回一个 IEnumerator 对象，该对象允许你遍历集合中的每个元素。

IEnumerator 接口

IEnumerator 接口定义了用于访问集合中元素的方法和属性。它有以下成员：

  • Current 属性返回集合中当前位置的元素。
  • MoveNext() 方法将迭代器移动到集合的下一个元素，并返回 true 如果有更多的元素可用，否则返回 false。
  • Reset() 方法将迭代器重置到集合的开头。

使用方法

使用 IEnumerable 和 IEnumerator 的一般模式是：

  1. 创建实现了 IEnumerable 接口的集合类。
  2. 在该集合类中，实现 GetEnumerator() 方法来返回一个实现了 IEnumerator 接口的迭代器。
  3. 使用 foreach 循环或手动调用 IEnumerator 的方法来遍历集合中的元素。

这样的设计允许你以一种通用的方式来迭代不同类型的集合，包括数组、列表、字典等。

总之，IEnumerable 和 IEnumerator 接口提供了一种通用的迭代机制，使得可以以一种一致的方式遍历不同类型的集合。

一些心得

  • IEnumerator本身是惰性的，在声明并赋值中并不触发生成本身的逻辑（如果有的话），只有在访问Current的时候才会去触发。

  • 没有执行MoveNext()就访问Current会进行报错，复合第一条，惰性特点。因为本身没有到达第一个元素。

  • 第一次moveNext()之后到达第一个元素。

  • 一个常见的类型想要实现Foreach的循环的条件-本质就是迭代器的实现。

    • 方案1.继承:IEnumerable然后实现GetEnumerator()来访问实际可以遍历的元素。
    • 方案2.手动实现IEnumerable.即手动建立GetEnumerator()方法返回一个IEnumerator的对象。在这个对象中继承IEnumerator并且实现MoveNext()，ReSet(),Current的方法与属性。
